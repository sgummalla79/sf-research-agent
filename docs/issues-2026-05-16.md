# Issues — 2026-05-16

---

## UI-001 — Selected model does not persist when conversation starts

**Page:** Chat

**Scenario:**
1. Selected a model
2. Sent Hello Message
3. Conversation started in full — selected model from step 1 is not persisting, changing to first model in the list

**Problem:**
Once the conversation starts, the model selection resets to the first model in the list — the model chosen in step 1 is not preserved.

**Expected:**
The model selected before sending the first message should remain active for the duration of the conversation.

---

## UI-002 — Chat name not reflected in side panel without page refresh

**Page:** Chat

**Scenario:**
1. Selected a model which is not the first one in the list
2. Sent Hello Message
3. Conversation started — chat gets a name, but the side panel does not update immediately
4. Name only appears in the side panel after manually refreshing the page

**Problem:**
The chat is being named correctly, but the sidebar does not reflect the name in real time. If there is any delay in the name being generated, the side panel stays stale until the user refreshes.

**Expected:**
The sidebar should update the chat name automatically once it is available, without requiring a page refresh.

---

## UI-003 — "Manage Skills" not visible when no skills are installed

**Page:** Chat

**Problem:**
When no skills are installed, the "Manage Skills" option does not appear, leaving the user with no way to navigate to skills and install one.

**Expected:**
"Manage Skills" should always be visible regardless of whether any skills are installed, so users can discover and install skills from an empty state.

# Improvements

## IMP-001 — Editable display name for model pills on Providers popup

**Page:** Settings → Providers → Provider popup

**Request:**
Model pills listed in the provider popup should have a pencil (edit) icon. Clicking it allows the user to update the display name of that model.

**Benefit:**
Users can rename models to friendlier or more recognizable labels without changing the underlying model ID.

---

## IMP-002 — Move Skills into Settings page as a dedicated tab

**Page:** Settings → (new Skills tab) / Chat

**Request:**
- Move the Skills section into the Settings page as a new tab in the left side pane, placed next to Providers
- When "Manage Skills" is clicked from the chat conversation message component, it should navigate the user directly to the Skills tab in Settings

**Benefit:**
Consolidates skill management under Settings where users already manage providers, and makes the "Manage Skills" shortcut from chat actually useful by deep-linking to the right tab.
